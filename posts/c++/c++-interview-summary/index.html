C++ Interview Summary

语法

1. Static关键字

  1. 面向过程

    （1）静态全局变量

    • 静态全局变量的作用域在当前cpp文件，可以在其他cpp文件中使用extern关键字声明；而全局变量的作用域在整个工程
    • 在全局数据区分配内存

    （2）静态局部变量

    • 只在首次执行到对象声明时初始化
    • 作用域在当前cpp文件，可以在其他cpp文件中使用extern关键字声明

    （3）静态函数

    • 作用域在当前cpp文件，可以在其他cpp文件中使用extern关键字声明

  2. 面向对象

    （1）静态成员变量

    • 作用域是类，而不是类的实例对象

    （2）静态成员函数

    • 作用域是类，而不是类的实例对象
    • 没有this指针，不能操作类的非静态成员函数

2. 指针和引用

  1. 指针
    • 指针是一个对象，值是一个地址，可以使用*取地址
    • 指针占用的内存（sizeof pointer）大小是4字节（32位机）/8字节（64位机）
    • 指针可以被const修饰，可以不赋初始值
    • 指针自增使指针指向下一个地址
  2. 引用
    • 引用是一个对象的别名，指向同一块内存
    • 引用占用的内存（sizeof reference）等于它指向的对象的内存
    • 引用必须被赋初始值
    • 引用自增使引用对象值加一

3. 多态

  1. 多态
    • 给一个基类指针赋予派生类的地址，使其呈现出多种形态和功能
  2. C++类的方法
    • 类的方法存储在固定的地址
    • 通过类对象指针调用类的方法时，编译器根据类型找到方法的地址
    • 编译时完成（静态联编）
  3. 纯虚函数
    • 纯虚函数 virtual = 0;
    • 对应的类叫做抽象类，不能被实例化
    • 是一个接口，其派生类必须实现这个接口，否则也不能被实例化
  4. 虚函数
    • 虚函数用virtual关键字修饰
  5. 虚函数表
    • 每个类都会维护一个虚函数表
    • 类对象被构造时，虚函数表被写到对象地址的初始位置
    • 对象执行虚函数时，从虚函数表取到函数地址，将this指针传递给函数并执行
    • 运行时完成（动态连编）
  6. 虚析构函数
    • 实现多态特性的时候，如果析构函数不是虚函数，指向派生类的对象会调基类的析构函数，造成内存泄漏
    • 将析构函数声明为虚函数，会先调用派生类的析构函数，再调用基类的析构函数

特性

1. C++ 11的特性

  1. 革新
    • decltype
      • 自动推导表达式类型
    • 可变参数列表
      • (T &t, …)
    • 右值引用
      • 消除两个对象交互时不必要的对象拷贝
      • 更简洁明确地定义泛型函数
    • move语义
      • 将传入的参数用static_cast强制类型转换为对应类型的右值
      • 返回（窃取）这个值
    • nullptr
  2. 语法糖
    • Lambda表达式
      • 匿名函数
      • []() {}
    • auto
      • 自动推导对象类型
    • 初始化参数列表
      • : x(), y(), … {}
    • 类的非静态成员函数在声明时赋值
    • 基于范围的for循环（Range-Based For Loop）
  3. 标准库扩充
    • 智能指针
      • unique_ptr, shared_ptr, weak_ptr
      • 弃用了auto_ptr
    • 哈希容器
      • unordered_map, unordered_map

2. 数据存储

  1. 栈区
    • 由编译器自动分配和释放
    • 存放参数，局部变量等
    • 分配方式类似于栈
  2. 堆区
    • 由开发者手动分配和释放，进程结束后由操作系统回收
    • 存放指针
    • 分配方式类似于链表
  3. 全局/静态存储区
    • 存放全局变量和静态变量
    • 已初始化的全局变量和静态变量存在一块地址，为初始化的存在另一块地址
    • 进程结束后由操作系统回收
